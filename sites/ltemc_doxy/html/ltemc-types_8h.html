◆ iop_t

typedef struct iop_tag iop_t

Struct for the IOP subsystem state. During initialization a pointer to this structure is reference in g_ltem1.

Each of the protocols (sockets, MQTT, HTTP) have unique behaviors: sockets receive asynchronous alerts but the data receive process is fully synchronous, HTTP is synchronous on receive tied to page read, MQTT is fully asynchronous with msg event receive signaling and data transfer taking place async behind the scenes initiated on interrupt.

◆ streamCtrl_t

typedef struct streamCtrl_tag streamCtrl_t

Abstract structure describing a data stream (protocol or filesystem).

The streamCtrl_t is used as an abstraction which is cast to a specific stream type EX: mqttCtrl_t* mqttCtrl = (mqttCtrl_t*)streamCtrl
