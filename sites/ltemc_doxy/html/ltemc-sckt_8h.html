◆ scktRecvFunc_t

typedef void(* scktRecvFunc_t) (streamPeer_t peerId, void *data, uint16_t dataSz)

Callback function for data received event. Notifies application that new data is available and needs serviced.

The *data and dataSz values are for convenience, since the application supplied the buffer to LTEmC.

Parameters
peerId[in] Data peer (data context or filesys)
handle[in] Subordinate data ID to give application information about received data
data[in] Pointer to received data buffer
dataSz[in] The number of bytes available

Open a data connection (socket) to d data to an established endpoint via protocol used to open socket (TCP/UDP/TCP INCOMING).

Parameters
sckt[in] - Pointer to the control block for the socket.
host[in] - The IP address (string) or domain name of the remote host to communicate with.
rmtPort[in] - The port number at the remote host.
lclPort[in] - The port number on this side of the conversation, set to 0 to auto-assign.
cleanSession[in] - If the port is found already open, TRUE: flushes any previous data from the socket session.
Returns
Result code similar to http status code, OK = 200
