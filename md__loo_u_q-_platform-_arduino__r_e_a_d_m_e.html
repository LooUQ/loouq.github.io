LooUQ - iotQc Tools and Library

The iotQc tools and library are provided to support LooUQ iotQc products. The library is written in c++ with the use of only C libraries for the purpose of portability, functionality, and readibility. The design schema is modular, so that the consumer can choose the level of complexity and control they wish to have when using iotQc products. LooUQ | iotQc

Supported Platforms and Required Tools

Platforms

  • Arduino (Zero, MKR1000, Feather M0, more to come...)
  • Windows IoT (Raspberry Pi 3, Dragon Board 410c, etc.)

Required Tools

  • Arduino IDE: for Arduino platform devices
  • Visual Studio 2015/2017: for Windows IoT platform devices

Additional Suggested Tools (Optional)

  • Atmel studio: for debugging purposes (Arduino Zero)
  • Visual micro extension: ease of use by integrating Arduino in Atmel/Visual studio.

Packages and Modules

The library is divided up into layers, making each layer its own independent module for use.

  1. Interface Layer
    • Generic Hardware interface
    • Spi interface
    • Serial interface
    • Bridge interface (Serial-Spi)
  2. Drivers Layer
    • Generic Driver (ABSTRACT)
  3. Transport Layer
    • Generic protocol
    • UDP
    • TCP/IP
    • HTTP
    • SSL (HTTPS)
  4. Platform Variants
    • <Platform> Spi
    • <Platform> Serial
    • <Platform> SSL session

The platform variants implement the layers via adapters. This provides an object oriented paradigm in the modules, and full platform optimization and functionality in the adapter while avoid preprocessor work. This allows for better modularity and readability.
